# Minishell Project

## Project Overview

This project is a C implementation of a simple shell, similar to `bash` or `zsh`. The goal is to parse user input, handle commands, and manage a basic shell environment.

**Core Technologies:**

* **Language:** C
* **Compiler:** `cc`
* **Build System:** `make`
* **Libraries:** The project uses a custom library `libft`.

**Architecture:**

The shell's architecture is based on a few key data structures:

* `t_shell`:  Manages the global state of the shell, including environment variables and the last command's exit code.
* `t_token`: Represents tokens (words, operators) generated by the lexer from the user's input.
* `t_command`: Represents a simple command, including its arguments and I/O redirections.

The general workflow appears to be:

1. **Lexing:** The input string is broken down into a series of tokens.
2. **Parsing:** The tokens are organized into a structure of commands.
3. **Execution:** The commands are executed, handling pipes, redirections, and built-in shell functions.

## Building and Running

The project uses a `Makefile` for building and management.

* **Build the project:**
  
  ```bash
  make
  ```
* **Run the shell:**
  
  ```bash
  ./minishell
  ```
* **Clean object files:**
  
  ```bash
  make clean
  ```
* **Full clean (objects and executable):**
  
  ```bash
  make fclean
  ```
* **Rebuild the project:**
  
  ```bash
  make re
  ```

## Development Conventions

* **Compiler Flags:** The project is compiled with `-Wall -Wextra -Werror`, enforcing a high standard of code quality.
* **Header Files:** Header files are located in the `includes` directory.
* **Source Files:** Source files are located in the `src` directory.
* **Library:** The `libft` library is located in the `lib` directory.
* **Object Files:** Object files are placed in the `obj` directory.
